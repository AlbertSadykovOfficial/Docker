

		Совместное использование данных

				Для совместного использования фалов хостом и одним или несколькими контейнерами,
				следует исползовать синтаксис:

						docker run -v <HOST_DIR>:<CONTAINER_DIR>

				К примеру, файлы конфигураций могут храниться на хосте и монтироваться внутри 
				контейнеров, создаваемых из однотипных образов. Так же можно совместно пользоваться
				данными в нескольких контейнера, если использовать ключ --volumes-from CONTAINER,

						Пример (доступ нового контейнера к томам контейнера в прдыдущем примере):

								docker run -it -h NEWCONTAINER --volumes-from container-test debian /bin/bash

								(Способ работает вне зависимости от того активен ли контейнер, содежращий тома)

				Том невозможно удалить пока существует хотя бы 1 контейнер, установивший связь с этим
				томом.


			Контейнеры данных

					На практике распространены контейнеры данных, цель которых - совместное использование
					двнных несколькими контейнерами.
					Преимущество - удобное пространство имен для томов, загружаемых через --volumes-from.

							Пример (создать контейнер данных СУБД PostgreSQL)

									docker run --name dbdata postgres echo "Data-only container for postgres"

									В дальнейшем мы можем пользоваться тамаи данного контейнера:

											docker run -d --volumes-from dbdata --name db1 postgres


			Удаление томов

					Тома удаляются, если

							- контейнер, сдржащий тома, был удален командой (docker rm -v)
							- в команде docker был включен флаг --rm, а также соблюдены следующие условия
									+ Отсутствие контейнеров, установивших связь с удаляемыми томами.
									+ Удаляемому тому не соответсвуе какой-либо каталог ФС хоста
										(при создании не использовали -v <HOST_DIR>:<CONTAINER_DIR>)

					Следует быть внимательными с удалением томов, потому что могут остаться потерянные файлы
					и каталоги (сироты-orphans) и будет сложно определить что они преставляют.
