

		Коммуникация с внешним миром

				Так как контейнеры изолированы, то их нужно уметь свяывать с другими приложениями.
				К примеру тот же web-сервер.

				Чтобы сделать проброс портов на контейнер, следует выполнить:

						docker run -d -p <HOST_PORT>:<CONTAINER_PORT> <APP>

						-p - перенаправить порт хоста на потр в контейнере.
						-P - автаматичесмкий проброс (поможет при большом кол-ве узлов)

						Пример:

								ID=$(docker run -d -P nginx)
								docker port $ID 80
									# Выведет ip+port
								curl ip:port
