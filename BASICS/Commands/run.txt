

		Команда run


				Команда запуска новых контейнеров

						Флаги:

								-a (--attach) - подключить поток (STDOUT) к терминалу
								-d (--detach) - запуск в режиме отключения от всех потоков
																запускает контейнер в фоновом режиме
																возвращает идентификатор (ID) контейнера

								-i (--interactive) - Поддерживает доступность открытого потока
																		 stdin, исп для запуска в интерактивном режиме
								--restart - позволяет настроить образ действий при попытке Docker 
														перезапуститьб остановленный контейнер.
														Агрументы:
															+ no - запрещает перезапуск
															+ always - всегда производить перезапуск
															+ on-failure - перезапуск для контейнеров,
																						 заврешивших работу не с 0 статусом
															+ on-failure:x; x - число попыток перезапуска
								-rm - автоматичекси удаляет контейнер после завершения сеанса работы.
											(несовместим с ключом -d)
								-t (--tty) - Создает терминал. (Обычно для интерактивного режима)

				Ключи для определения имен контейнеров и внутренних переменных среды.

						-e (--env) - определить переменные среды внутри контейнера.

													docker run -e var1=val -e var2="val 2" debian env

							 (--env-file) - передача переменных черех файл.

						-h (--hostname) - устанавливает для запускаемого контейнера заданное имя Unix-хоста.

													docker run -h "myhost" debian hostname

						--name <NAME> - приваивает контейнеру имя NAME


				Создание и настройка томов

						-v (--volume)
						--volumes-from - часто \используют при работе с котейнерами данных

				
				Создание сетевой среды:

						--expose - аналог EXPOSE Dockerfile, определяет номер порта или диапазон портов,
											 предназначенны для использования, но не открывает их.
											 Целесообразно использовать с ключом -P
						--link - настраивает интерфей частной закрытой сети длязаданного контейнера
						-p (--publish) - делает порт достуупным с хоста.
						-P (--publish-all) - объявляет все порты, открываемые в контейнере, доступными на хосте.

				Змена параметров настройки, определенных в Dockfil

						--entrypoint - определяет точку входа для запускаемого контейнера в соответствие с 
													 заданным аргументов, заменяя инструкцию ENTRYPOINT Dockerfile.

						-u (--user) - Определяет пользователя, от имени которого выполня.тся команды.
													(Может быть как символьное имя, так и UID)

						-w (--workdir) - Устанавливает рабочий каталог в контейнере в соответствии с заданным
														 путевым именем. Заменяет люые значения в Dockerfile.
