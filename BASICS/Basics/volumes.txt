

		Управление данными с помощью томов и контейнеров данных

				Тома - каталоги, которые не являются частью системы UnionFS контейнера,
							могут быть смонтированы как отдельные файловые системы хоста.


				Существует 3 способа инициализации томов:

						1) При запуске контейнера через флаг -v

								Пример (сделать каталог data внутри контейнера томом):

										docker run -it --name conteainer-test -h CONTEAINER -v /data debian /bin/bash

										Проверить расположение в ФС:

												docker instpect -f {{.Mounts}} container-test

						2) Объявление с помощью инструкции VOLUME внутри Dockerfile

									Пример:

											FROM debian:wheezy
											VOLUME /data

									При этом различные операции с томами (такие как установка прав доступа)
									следует делать до инструкции VOLUME, иначе они будут применяться к 
									временному контейнеру.

						3) Расширение аргумента -v явным укаанием связываемого каталога хоста в формате
							 -v HOST_DIR:CONTAINER_DIR.

							 		Пример (монтируем каталог ФС хоста /home/adrian/data как /data в контейнере)
							 					 (если каталог data имел сожержимое ранее, оно будет скрыто):

											 docker run -v /home/adrian/data:/data debian ls /data
