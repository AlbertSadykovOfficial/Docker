

		Автоматические сборки.


				Автоматические сборки конфигурируются через сайт Docker Hub.

						1) Create
						2) Create Automated Build
						3) Указываем расположение репозитория на Github
						4) Изменяем по нужде имя репозитория (к примеру identidock_auto)
						5) Добавляем краткое описание (рекомендуется)
						6) Вводим Branch -> master
						7) В поле Dockerffile Location указываем расположение (identidock/Dockerfile)
						8) Присваиваем имя образа

				Для проверки работы можно добавить файл README в каталог identidock
				Затем следует отправиьт его на github^

						git add README.md
						git commit -m "Added README"
						git push

						(Ждем некоторое время, переходим на сайт сборки образа и наблюдаем результат)

				Если по какой-то причине в процессе сборки возникла ошибка, то можно попытаться 
				выяснить ее причину по записям в журнале (Build Code) на вкладке (Build Details)
				Можно начать новую процедуру сборки Trigger a Build


				Docker Hub подойдет для небольших проектов, для более серьезных проектов потребуются
				другие решения
