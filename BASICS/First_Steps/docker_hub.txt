

		Работа с реестрами

				Все работы лежат на Docker Hub и именно оттуда скачиваются компоненты,
				мы как пользователи так же можем загружать туда проекты.

				Поиск образов осуществляется через команду поиска Docker или непосредственно
				на сайте http://registry.hub.docker.com

				Реестр (registry) - сервис, отвечающий за хранение образов (Docker Hub)
				Репозиторий - набор взаимосвязанных образов (к примеру, различные версии 1го приложения)
				Тег - ID, присваиваемый образам внутри репозитория.

				Пример: docker pull amouat/revealjs:latest - загрузит образ с тегом latest в репозиторий
								amouat/revealjs из реестра Docker Hub

				(Чтобы выгружать свои реестры, нужно зарегистриоваться на Docker Hub)

				Пространство имен образов:

						Реестр Docker Hub:
							Имена типа debian или ubuntu - оффициальные и широкораспространенные программы
							Имена типа (/) amount/revealjs - Прострастов имен "user" - образы, выгруженные
																							 конкретными пользователями.
						Сторонние реестры:
							Имена с префиксами хоста или IP - реестры вне Docker Hub


				Оффициальные репозитории не содержат префиксов как было сказано ранее,
				либо содержат преикс library : library/mongo

				Чтобы поместить в репозиторий контейнер, следует выполнить команду login,
				затем выполнить команду pull <project>

						sudo docker login
							...(username password)
						sudo docker push albertsadykovofficial/cosway

						К имени репозитория можно добавить тег:

								sudo docker push albertsadykovofficial/cosway:latest
