

		Включение модульных тестов

				Код тестов включается в кодовую базу приложения,
				обычно распологаясь в каталоге app, где и само приложение.

				Тесты будут проверять основную функциональность главного
				приложения, без затрагивания модулей (test.py)

				Наше старое приложение не пройдет проверку, потому что 
				при вводе данных через input-поле мы не проверяем их 
				на корректность и не экранируем, поэтому можно встроить
				инъекцию в запрос.

				Чтобы устранить эту проблему нужно в исходном коде добавить
				функцию, которая обрабатывает запрещенные символы:

							name = html.escape( request.form['name'], quote=True )

							Метод escape() приводит данные к нужной форме.


				После написания теста и исправления кода, можно создать приложение

							docker build -t identidock .
							docker run identidock python test.py

				
				Чтобы тесты автоматически выполнялись, следует добавить режим 
				тестирования к существующему режиму разработки в cmd.sh:

						if "$ENV" = 'UNIT'; then
								echo "Running Unit Tets"
								exec python "test.py"

				После написания теста и исправления кода, можно создать приложение

							docker build -t identidock .
							docker run -e ENV=UNIT identidock
